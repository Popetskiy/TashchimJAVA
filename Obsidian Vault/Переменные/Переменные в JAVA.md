## summary

Переменная предоставляется нам как место хранения наших данных (или адрес места) 
У переменных примитивного типа есть определенный формат данных, размер, диапазон значений, которые могут храниться в памяти. 

Константа — это заданные, фиксированные значения, которые не должны меняться 

### Место хранения и адреса

Например  компьютерная память — это **огромный склад** с миллионами пронумерованных ячеек. [[Память в JAVA]]

Вот так  это работает на пальцах:

1. **Адрес памяти:** Каждая ячейка на этом складе имеет свой длинный и сложный номер (например, `0x7ffcd4`). Роботу-процессору удобно работать с такими цифрами, а человеку — нет.
2. **Переменная:** Это просто **понятный ярлык (имя)**, который ты вешаешь на эту ячейку. Вместо того чтобы запоминать «положи число 5 в ячейку №00451», говорим: «пусть в переменной `score` лежит 5».
3. **Место хранения:** Когда  создается переменная, компьютер выделяет  «коробку» нужного размера. Можешь класть туда данные, вынимать их или заменять на новые.

**Итог:** Переменная — это удобный псевдоним для конкретного места в памяти, чтобы можно было обращаться к своим данным по имени, а не по «прописке».
## Виды переменных

- переменные объекта <font color="#9bbb59"> (называются полями объекта)</font> - Переменные создаются лишь после того, как будет создан объект (то есть после создания объекта с помощью `new`). В объекте внутренние переменные всегда видимы для всех методов, конструктора или чего-либо внутри этого же объекта.

- локальные переменные - Данные переменные объявляются в методах, конструкторах или блоках . Они видимы лишь в пределах объявленного метода, конструктора или блока. То есть использовать переменную, например, в другом методе нельзя.

- переменные класса <font color="#9bbb59">(называются поля класса)</font> -данный вид переменных ещё называют статическими. Они объявляются со словом-модификатором `static`, но за пределами метода, конструктора или блока. [[Статическая переменная]]

 <font color="#9bbb59">**Могут быть не только static**:</font>
 Слово **«Поле» (Field)** — это общее название для любой переменной, объявленной внутри класса (вне методов). Поля делятся на две категории:

1. **Поля экземпляра (Instance Fields):** Объявлены _без_ `static`. У каждого объекта (экземпляра) будет своя копия такой переменной.
2. **Поля класса (Class Fields / Static Fields):** Объявлены _с_ `static`. Одна копия на всех.

Существует особый вид переменных — константы, которые могут быть глобальными, переменными. Константа — это заданные, фиксированные значения, которые не должны меняться. Их область видимости — вся программа.
Имя константы прописывается в верхнем регистре и не в camelCase, а через ** _ **
LIMIT_SUM

## **Кратко**
переменная экземпляра — хранит данные конкретного объекта;
локальная переменная — существует только внутри метода или блока;
переменная класса (static) — общая для всех объектов;
константа (final, часто static final) — неизменяемое значение.
