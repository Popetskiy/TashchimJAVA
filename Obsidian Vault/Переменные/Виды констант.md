В Java нет «глобальных» переменных в том смысле, как в C++ или Python (где переменную можно объявить просто в файле вне всего). В Java всё находится внутри классов.

Поэтому под «глобальными» в Java обычно понимают статические переменные, а под «не глобальными» — переменные экземпляра или локальные переменные.

## 1. Глобальные (Статические) константы

Они доступны всей программе (если они `public`). Они создаются один раз и живут в памяти класса.

```java
public class Config {
    // Глобальная константа: доступна везде через Config.VERSION
    public static final String VERSION = "1.0.2";
    
    // Еще одна: доступна везде через Config.MAX_SCORE
    public static final int MAX_SCORE = 9999;
}
```

## 2. Не глобальные константы (Константы объекта)

Такая константа принадлежит конкретному объекту. У каждого созданного объекта она своя, и её нельзя изменить после создания.

```java
class User {
    // Константа объекта: у каждого пользователя свой ID, 
    // который нельзя поменять после присвоения в конструкторе.
    public final int id;

    public User(int id) {
        this.id = id;
    }
}
```

## 3. Локальные константы

Они живут только внутри одного метода. Как только метод отработал, переменная исчезает.

```java
public void calculate() {
    // Локальная константа: видна только внутри этого метода
    final double TAX_RATE = 0.13;
    
    double result = 100 * TAX_RATE;
    System.out.println(result);
}
```

---

## В чем разница простыми словами:

|Вид константы|Где живет?|Как обратиться?|Когда умирает?|
|---|---|---|---|
|Глобальная (`static final`)|В классе|`Класс.КОНСТАНТА`|Когда закроется программа|
|Объекта (`final`)|В объекте|`объект.константа`|Когда удалят объект из памяти|
|Локальная (`final` в методе)|В методе|Просто по имени|Когда метод завершится|
