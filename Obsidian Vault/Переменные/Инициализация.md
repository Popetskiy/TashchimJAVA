Инициализация — это ==процесс присвоения переменной её первого (начального) значения==.

Если «объявление» переменной — это просто создание пустой подписанной коробки (`int score;`), то «инициализация» — это момент, когда  впервые кладешь в неё предмет (`score = 10;`).

Что касается переменных объекта (их еще называют полями класса), тут есть важные правила:

## 1. Жизненный цикл (Когда они появляются?)

Переменные объекта не существуют «сами по себе». Они — часть структуры объекта.

- Пока  просто написан код класса, памяти под переменные нет.
- Как только  написать `new MyClass()`, Java выделяет в Куче (Heap) блок памяти, достаточный, чтобы вместить все переменные этого объекта.
- Инициализация происходит именно в этот момент.

## 2. Автоматическая (дефолтная) инициализация

В отличие от локальных переменных (внутри методов), переменные объекта в Java всегда инициализируются автоматически, если  не задать им значение вручную:

- Числа (`byte`, `int`, `double` и т.д.) → `0` или `0.0`
- `boolean` → `false`
- `char` → `\u0000` (пустой символ)
- Ссылочные типы (объекты, строки) → `null` (пустота, «адреса нет»)

## 3. Где можно инициализировать переменные объекта?

Есть три способа дать им значение:

1. При объявлении: `int age = 18;` (сработает сразу при создании объекта).
2. В конструкторе: `public MyClass() { age = 20; }` (самый гибкий способ).
3. В блоке инициализации: `{ age = 25; }` (выполняется перед конструктором).

Простая аналогия:  
Представим чертеж автомобиля (это Класс). В чертеже написано, что у машины должен быть «цвет». Но пока  не собрать на заводе реальную машину (Объект с помощью `new`), нет физического места, которое можно покрасить. В момент сборки машина получает свой цвет (происходит инициализация).
