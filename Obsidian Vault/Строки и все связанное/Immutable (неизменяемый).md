Immutable (неизменяемый) — это объект, ==состояние которого нельзя изменить после того, как он был создан==.

Если тебе нужно что-то поменять в таком объекте, ты не правишь старый, а просто создаешь новый с другими данными.

## Самый простой пример: String

Строки в Java — классический пример immutable-объекта.

```java
String s = "Привет";
s.toUpperCase(); // Метод возвращает НОВУЮ строку "ПРИВЕТ"
System.out.println(s); // Напечатает всё равно "Привет", оригинал не изменился
```

## Как сделать свой класс Immutable?

Чтобы объект стал «непробиваемым», нужно соблюсти 4 правила:

1. Сделать класс `final` (чтобы никто не создал «хитрого» наследника, который сможет менять данные).
2. Сделать все поля `private` и `final` (чтобы к ним нельзя было подобраться снаружи и нельзя было перезаписать внутри).
3. Не делать сеттеры (методы `setAge`, `setName` и т.д.).
4. Инициализировать всё только через конструктор.

## Зачем это нужно? (3 причины)

1. Безопасность: Ты точно знаешь, что если ты передал объект в другую часть программы, его там никто «втихую» не подправит.
2. Многопоточность: Неизменяемые объекты идеальны для работы в несколько потоков. Им не нужны «замки» (synchronized), потому что данные всё равно никто не может изменить — значит, и конфликтов не будет.
3. Кэширование: Java может спокойно хранить такие объекты в памяти (как в пуле строк), зная, что они не изменятся.
