[[Что такое класс]]



В Java функции принято называть методами. ==Это именованные блоки кода, которые решают конкретную задачу и могут использоваться в программе многократно==. 
[[Частью чего функция может быть, а чего не может быть]]
## Что это такое?

Метод — это «подпрограмма», которая объединяет несколько строк кода под одним именем. Вместо того чтобы переписывать одни и те же действия (например, вычисление суммы или вывод текста), вы один раз описываете их в методе, а затем просто «вызываете» его по имени!

## Как устроена функция (метод)

Стандартная структура метода выглядит так: [4]

```java
public static int sum(int a, int b) {
    return a + b;
}
```

- Модификаторы (`public static`): определяют доступ к методу и его тип. [[Модификаторы доступа (private, public, protected, Default)|Модификаторы]]
- Тип возвращаемого значения (`int`): какой результат выдаст функция (число, текст или `void`, если функция ничего не возвращает). [[Переменные примитивного типа|Переменные]]
- Имя метода (`sum`): по нему вы будете обращаться к функции.
- Параметры (`int a, int b`): входные данные, которые функция принимает для работы. - это [[Сигнатура функции(метода)]]
- Тело метода `{...}`: сам код, который выполняется.
- Оператор `return`: команда, которая завершает работу метода и возвращает результат. 
## Как работают функции

1. Объявление: вы создаете метод внутри класса, описывая, что он должен делать. [[Что такое класс#Класс в Java|Класс это]]
2. Вызов: когда программе нужно выполнить задачу, вы пишете имя метода и передаете ему нужные данные в скобках (например, `sum(5, 10)`).
3. Выполнение: программа временно переходит в тело метода, выполняет его код и, если нужно, возвращает результат обратно в то место, откуда был сделан вызов. 

## Зачем они нужны

- Повторное использование: написали один раз — используете везде.
- Читаемость: код становится чище, так как сложные задачи разбиваются на мелкие понятные шаги.
- Легкость правок: если нужно изменить логику вычислений, вы правите её только в одном месте (внутри метода), а не во всей программе. 

## Статичный метод отличия
 1. Обычные методы (Методы объекта)

Это то, что умеет делать конкретный дом. Например, «открыть дверь» или «включить свет».

- Чтобы включить свет, тебе нужен конкретный адрес (экземпляр дома). Ты не можешь включить свет «вообще во всех чертежах».
- В коде это выглядит так: сначала создаешь дом через `new`, а потом вызываешь метод: `мойДом.включитьСвет()`.

 2. Статические методы (Методы класса)

Это то, что относится к чертежу или к понятию «дом» в целом. Например, «посчитать общую стоимость кирпича» или «узнать правила застройки».

- Тебе не нужно строить дом, чтобы узнать эти правила. Ты просто смотришь в документы (обращаешься к названию класса).
- В коде это выглядит так: ты сразу пишешь имя класса: `Дом.узнатьПравила()`.

## Главная разница на пальцах:

|Признак|Обычный метод|Статический метод (`static`)|
|---|---|---|
|К чему привязан?|К конкретной "штуке" (объекту)|К самому названию (классу)|
|Нужен ли `new`?|Да, сначала создаем объект|Нет, вызываем напрямую|
|Что видит?|Видит данные объекта (свой цвет, свой размер)|Видит только общие данные (константы)|
|Пример из жизни|`твояГитара.играть()`|`Гитара.настроитьПоТюнеру()`|

Когда какой использовать?

- Если методу нужно знать параметры конкретного объекта (например, его вес или имя) —  обычный.
- Если метод — это просто «инструмент» или «формула», которой всё равно, какой там объект (например, расчет корня числа или перевод текста в капслок) — статический.
## Пример
```
public class Cat {
    String name; // Имя конкретного кота
    static int catCount = 0; // Общий счетчик для всех котов в мире

    public Cat(String name) {
        this.name = name;
        catCount++; // При создании нового кота увеличиваем общий счетчик
    }

    // ОБЫЧНЫЙ МЕТОД: Кот мяукает (нужен конкретный кот)
    public void meow() {
        System.out.println(name + " говорит: Мяу!");
    }

    // СТАТИЧЕСКИЙ МЕТОД: Узнать, сколько всего котов создано
    public static void showTotalCats() {
        // Не знаем имен, просто знаем число
        System.out.println("Всего котов: " + catCount);
    }

    public static void main(String[] args) {
        // 1. Вызываем статический метод напрямую через класс (объектов еще нет)
        Cat.showTotalCats(); 

        // 2. Создаем объекты (конкретных котов)
        Cat myCat1 = new Cat("Барсик");
        Cat myCat2 = new Cat("Мурзик");

        // 3. Вызываем обычные методы у объектов
        myCat1.meow(); // Барсик говорит: Мяу!
        myCat2.meow(); // Мурзик говорит: Мяу!

        // 4. вызываем статический метод
        Cat.showTotalCats(); // Всего котов: 2
    }
}
```

---

Итог для «чайника»:  
Метод в Java — это как комната. Она может быть только внутри дома (класса). Ты не можешь построить комнату прямо внутри другой комнаты или просто посреди чистого поля без фундамента.
