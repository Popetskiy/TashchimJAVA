 функция (метод) — это розетка.

## Где функция ОБЯЗАНА быть (ее «фундамент»)

В Java функция может жить только внутри Класса.

- Класс — это как стена дома. Ты не можешь просто повесить розетку в воздухе посреди поля. Тебе нужна стена (Класс), в которую ты её вмонтируешь.
- _Код:_ `class СТЕНА { void розетка() { ... } }` — Правильно.
- _Код:_ `void розетка() { ... }` (просто в файле) — Ошибка.

## Частью чего функция НЕ МОЖЕТ быть (где её нельзя «лепить»)

1. Внутри другой функции.  
    Нельзя  вмонтировать одну розетку прямо _внутрь_ другой розетки. В Java нельзя объявить метод внутри другого метода.
    
    - _Ошибка:_ `void метод1() { void метод2() { ... } }`
    
2. Внутри циклов или условий.  
    Нельзя начать «строить» розетку прямо во время того, как работает телевизор (`if` или `for`).
    
    - _Ошибка:_ `if (условие) { void метод() { ... } }`
    
3. Сама по себе (Глобально).  
    В некоторых языках (как C++ или Python) можно бросить функцию где угодно. В Java — никогда. Только внутри «стены» (Класса).

---

**Итог:**  
Функция в Java — это всегда часть Класса. Она не может быть «бездомной» и не может «матрешкой» сидеть внутри другой функции.
