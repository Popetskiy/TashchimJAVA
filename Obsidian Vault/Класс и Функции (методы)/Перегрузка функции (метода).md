Перегрузка методов (overloading) — ==это когда ты создаешь в одном классе несколько функций с одинаковым именем, но с разными входными данными (разными сигнатурами)==.

## Для чего это нужно? 

Представим, что  есть метод `рисуй()`.

-  хочу нарисовать круг — даю методу только радиус: `рисуй(5)`.
-  хочу нарисовать прямоугольник — даю длину и ширину: `рисуй(10, 20)`.
-  хочу нарисовать точку — вообще ничего не даю: `рисуй()`.

Вместо того чтобы придумывать кривые имена типа `рисуйКруг()`, `рисуйПрямоугольник()` и `рисуйТочку()`,  называем их одинаково. Java сама поймет, какую версию вызвать, по тому, что именно ты положил в скобки.

---

## Как это выглядит в коде:

```java
public class Printer {
    // Вариант 1: Печатаем число
    void print(int number) {
        System.out.println("Я печатаю число: " + number);
    }

    // Вариант 2: Печатаем текст (имя то же, данные другие)
    void print(String text) {
        System.out.println("Я печатаю текст: " + text);
    }
}
```

## Зачем это делать?

1. Удобство:  не нужно запоминать 10 разных названий для одного и того же действия (например, метод `System.out.println` перегружен — он умеет печатать и числа, и буквы, и объекты, но имя всегда одно).
2. Чистота кода: Программа выглядит логичнее, когда похожие действия называются одинаково.
3. Гибкость:  можно использовать метод в разных ситуациях, подсовывая ему разные данные.

## Важное правило:

Чтобы перегрузка заработала, у методов должны быть разные параметры (либо другое количество, либо другие типы). Если просто поменять `return int` на `return String`, но оставишь те же скобки — Java выдаст ошибку.

## Перегрузка статичных методов

[[Статичные методы#- Статические методы| Стат метод Это]]

Для Java нет разницы, стоит перед методом слово `static` или нет — правила перегрузки для них одинаковы. Главное, чтобы у методов были разные сигнатуры (разные наборы параметров в скобках).

## Пример перегрузки статических методов:

```java
public class Calculator {
    
    // Статический метод для целых чисел
    public static int add(int a, int b) {
        return a + b;
    }

    // Перегруженный статический метод для дробных чисел
    public static double add(double a, double b) {
        return a + b;
    }

    // Перегруженный статический метод для ТРЕХ чисел
    public static int add(int a, int b, int c) {
        return a + b + c;
    }

    public static void main(String[] args) {
        // Java сама выберет нужный статический метод по аргументам:
        System.out.println(Calculator.add(5, 10));          // Вызовет первый
        System.out.println(Calculator.add(2.5, 3.5));      // Вызовет второй
        System.out.println(Calculator.add(1, 2, 3));       // Вызовет третий
    }
}
```

## Есть ли ограничения?

Единственное важное ограничение: нельзя создать два метода с одинаковой сигнатурой, где один `static`, а другой — нет.

- _Пример ошибки:_

    ```java
    void print(int a) { ... }
    static void print(int a) { ... } // ОШИБКА! Сигнатуры print(int) совпадают.
    ```
  Java запутается: «Ты вызвал `print(5)`, а мне какой запустить — статический или обычный?».

Зачем перегружать статические методы?  
Обычно это делают в утилитных классах (как `Math`). Например, метод `Math.abs()` (модуль числа) перегружен для `int`, `long`, `float` и `double`, чтобы ты мог передать туда любое число и получить результат.
