==Это класс, который запрещено наследовать.==

Если  пометить класс словом `final`,  говорим Java: «Этот чертеж идеален, никто не может его дополнить или изменить через создание подклассов».

## В чем разница:

- Константа (final переменная) — это запрет на изменение значения. (Например, `final int AGE = 25;` — число 25 нельзя поменять на 26).
- Final класс — это запрет на наследование. У этого класса не может быть «детей» (extends).

## Пример из жизни:

Самый известный `final` класс в Java — это `String`.  
Разработчики Java сделали его `final`, чтобы никто не мог создать свой класс «СуперСтрока», который мог бы сломать логику работы с текстом или нарушить безопасность.

## Как это выглядит в коде:

```java
final class Animal { 
    // ... какой-то код
}

// ОШИБКА! Компилятор не даст это сделать
class Dog extends Animal { 
}
```

## Короткий итог:

1. `final` переменная = нельзя менять значение (это и есть константа).
2. `final` метод = нельзя переопределить в наследниках.
3. `final` класс = нельзя создавать от него наследников.
