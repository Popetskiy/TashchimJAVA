Полиморфизм в Java — это ==способность программы использовать объекты с одинаковым интерфейсом без информации о конкретном типе этого объекта==.

Если совсем просто: это когда одна команда выполняет разные действия в зависимости от того, на каком объекте она вызвана.

## Как это работает (на примере):

Представим, что есть общий класс `Animal` с методом `makeSound()`. От него наследуются `Dog` и `Cat`.

```java
Animal myPet = new Dog(); 
myPet.makeSound(); // Выведет "Гав!"

myPet = new Cat();
myPet.makeSound(); // Тот же самый вызов команды, но теперь выведет "Мяу!"
```

## Почему это круто?

 Не нужно знать, кто именно сейчас находится в переменной `myPet`. Просто говоришь: «Издай звук!», и объект сам «знает», как это сделать.

## Два вида полиморфизма в Java:

1. Динамический (Overriding / Переопределение):  
    Это то, что в примере выше. Когда у предка и наследника методы называются одинаково, но делают разное. Java решает, какой метод запустить, прямо во время работы программы.
2. Статический (Overloading / Перегрузка):  
    Это когда в одном классе есть несколько методов с одинаковым именем, но разными параметрами.  
    Например: `print(int i)` и `print(String s)`. Java еще при написании кода понимает, какой из них вызвать, глядя на то, что ты передал в скобки.

## Итог простыми словами:

Полиморфизм позволяет тебе писать универсальный код. Ты можешь создать список «Животных» и в цикле заставить их всех «подать голос», не разбираясь, кто там собака, а кто кошка.
