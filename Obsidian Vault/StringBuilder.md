`StringBuilder` — это специальный класс в Java, предназначенный для создания и изменения строк.

Главное отличие от обычного `String`: объект `StringBuilder` можно изменять (он mutable).

## 1. Зачем он нужен?

 Обычные строки (`String`) нельзя менять. Если вы в цикле 100 раз прибавите символ к строке через `+`, Java создаст 100 новых объектов в памяти. Это медленно и тратит ресурсы.

`StringBuilder` работает иначе: он создает один внутренний буфер (массив символов) и просто дописывает в него данные, не создавая гору лишних объектов.

## 2. Основные методы

У `StringBuilder` свои методы для модификации текста:

- `append(value)` — добавляет в конец (число, строку, символ).
- `insert(index, value)` — вставляет данные в середину строки.
- `delete(start, end)` — удаляет кусок текста.
- `reverse()` — разворачивает строку задом наперед (классика на собеседованиях).
- `toString()` — превращает накопленный результат в обычный `String`.

## 3. Пример кода

```java
StringBuilder sb = new StringBuilder("Кот");

sb.append(" Бегемот");      // Теперь в буфере: "Кот Бегемот"
sb.insert(3, " черный");   // Теперь: "Кот черный Бегемот"
sb.reverse();              // Развернули задом наперед

String result = sb.toString(); // Превратили в обычную строку для работы
```

## 4. StringBuilder vs StringBuffer

- `StringBuilder`: Работает очень быстро, но небезопасен в многопоточной среде. Если два потока одновременно начнут в него писать, данные могут перемешаться или возникнет ошибка. Используется в 99% случаев в обычном коде.
- `StringBuffer`: Полный аналог, но потокобезопасен (Thread-safe). Он медленнее, так как тратит время на «блокировки» для потоков.

Вкратце:  
Если нужно собрать строку из кусочков в цикле — берите `StringBuilder`. Если нужно просто хранить текст и сравнивать его — оставляйте `String`.
