**Модификаторы доступа** — это ключевые слова, которые определяют «границы видимости» ваших классов, методов и переменных. Это основа инкапсуляции (скрытия внутренней логики).
## 1. public (Публичный)

- Доступ: Откуда угодно. Из любого пакета и любого класса в проекте.
- Аналогия: Главный вход в торговый центр.
- Пример: `public class Cat` или `public void meow()`.

## 2. private (Приватный)

- Доступ: Только внутри того же класса, где он объявлен. Даже наследники (другие классы-коты) не увидят эти данные.
- Аналогия: Личный сейф в вашей квартире.
- Пример: `private int age;` (мы скрываем возраст, чтобы его нельзя было изменить случайно извне без проверки).

## 3. protected (Защищенный)

- Доступ: Внутри того же пакета + во всех классах-наследниках (даже если они в другом пакете).
- Аналогия: Семейная реликвия, доступная только родственникам.
- Пример: `protected String breed;` (порода видна всем «детям» класса Cat).

## 4. Default (По умолчанию / Package-private)

- Как задается: Если вы не написали никакой модификатор.
- Доступ: Только внутри того же пакета.
- Аналогия: Закрытый двор жилого комплекса — только для «своих» соседей по пакету.

## Зачем это нужно в тестировании и разработке?

1. Безопасность данных: Мы делаем поля `private` и работаем с ними через специальные методы (геттеры и сеттеры), чтобы никто не записал в `age` отрицательное число.
2. Чистота API: Когда вы тестируете чужой класс, вам видны только `public` методы. Это упрощает работу — вы видите только то, что «выставлено наружу» для использования.
