Тестирование программного обеспечения классифицируется по нескольким ключевым признакам: методам (подходам), уровням и видам.

## 1. Методы тестирования (по доступу к коду)

Методы определяют, насколько глубоко тестировщик должен знать внутреннюю логику программы: 

- Метод «Черного ящика» (Black Box): Тестирование только через пользовательский интерфейс, без доступа к исходному коду. Проверяется соответствие результата ожиданиям при определенных входных данных.
- Метод «Белого ящика» (White Box): Тестирование с полным доступом к коду, архитектуре и внутренней структуре программы. Часто выполняется разработчиками (например, Unit-тесты).
- Метод «Серого ящика» (Grey Box): Комбинированный подход, где тестировщик имеет частичный доступ к коду или базе данных для более глубокой проверки сценариев. 

## 2. Уровни тестирования

Отражают иерархию процесса разработки — от малого к целому: 

1. Модульное (Unit): Проверка отдельных минимальных компонентов (функций, методов).
2. Интеграционное: Проверка взаимодействия между несколькими модулями или системами.
3. Системное: Тестирование всего программного продукта в сборе на соответствие исходным требованиям.
4. Приемочное (Acceptance): Финальный этап, подтверждающий готовность продукта к выпуску и его соответствие нуждам заказчика. 

## 3. Основные виды тестирования

Виды тестирования группируются по целям проверки: 

- По функциональности:
    
    - Функциональное: Проверка того, _что_ делает система (выполняет ли она свои задачи).
    - Нефункциональное: Проверка того, _как_ работает система (производительность, безопасность, удобство использования).
    
- По времени проведения:
    
    - Регрессионное: Повторная проверка после изменений в коде, чтобы убедиться, что старый функционал не сломался.
    - Дымовое (Smoke): Поверхностная проверка самых важных функций после сборки («жива» ли программа вообще).
    - Санитарное (Sanity): Быстрая проверка конкретной функции после исправлений.
    
- По степени автоматизации:
    
    - Ручное: Выполняется человеком вручную по тест-кейсам.
    - Автоматизированное: Выполняется с помощью специальных инструментов и скриптов.
    
- По состоянию системы:
    
    - Статическое: Анализ кода, документации и дизайна без запуска программы.
    - Динамическое: Тестирование работающего приложения в процессе его выполнения.
